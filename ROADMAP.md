# Roadmap

MdPreview 的目标是成为一个稳定、低干扰、离线优先的 Windows 本地 Markdown 编辑器。

## 近期目标

### Windows 打开体验

- 稳定支持 `.md`、`.markdown`、`.mdown`、`.mkd` 文件关联
- 优化从系统“打开方式”启动时的文件载入流程
- 处理应用已运行时再次打开文件的窗口聚焦和文档切换
- 增加文件占用、路径异常和编码异常提示

### 编辑体验

- 优化大纲定位和长文档滚动体验
- 改进未保存变更提示
- 优化粘贴图片后的路径管理
- 增加最近打开文件入口
- 改进空文档、无标题文档和大文档的状态提示

### Markdown 兼容

- 补充表格、任务列表、代码块、图片、链接等常见内容回归样例
- 处理中文文档、长标题、深层标题和特殊字符标题
- 增强导出 HTML 的样式一致性

## 中期目标

### 质量体系

- 增加单元测试和端到端打开文件验证
- 增加 release checklist
- 增加 GitHub Actions 构建验证
- 建立 Windows 打包问题排查文档

### 安全边界

- 审查 Electron 主进程和 preload API 暴露面
- 收紧本地文件访问边界
- 明确远程图片、导出 HTML 和剪贴板图片处理规则
- 为安全相关变更增加 review checklist

### 发布体验

- 完善安装版和便携版说明
- 增加版本变更日志
- 改进自动更新或手动升级指引
- 提供基础截图和演示素材

## 长期目标

- 提供稳定的本地 Markdown 写作体验
- 建立可维护的插件或扩展边界
- 支持更多导出格式
- 提升大文件和长文档性能
- 根据社区反馈评估 macOS/Linux 支持

## 非目标

MdPreview 暂不计划成为：

- 云端协作文档平台
- 重型知识库系统
- 带账号体系的笔记服务
- 在线 Markdown 托管服务

项目会优先保持本地、轻量、可维护。
